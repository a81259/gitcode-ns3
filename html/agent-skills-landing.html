Agent Skills

Agent Skills 是随代码迭代演进的规则文件。支持这套机制的 AI 工具在打开仓库后，就能直接读取规则、调用 tools、复用知识卡，再按项目事实给建议。

Quick Start

打开仓库后，三步开始用 skills

重点不是先学一套新命令，而是直接在仓库里提问题、给目标、看结果。支持这套机制的 AI 工具会顺着仓库内的技能文件和规则继续往下做。

1

打开仓库

用支持规则和 skills 的 AI 工具打开这个仓库。进入仓库后，相关技能文件、知识卡和规则就能接入上下文。

2

直接说目标

不用先背目录。直接说你想跑什么、比较什么、哪里看不懂，例如想改 case、查参数、读结果或理解协议行为。

3

让 skills 接着做

后续会按仓库规则继续推进：该调 tools 就调 tools，该复用知识卡就复用知识卡，需要时再回到源码、参数和结果文件核对。

可以直接这样开始：我想比较 CBFC 和 PFC 的队列行为、帮我看看这个 case 该从哪里改、这个结果为什么吞吐掉下来了。

如何开始

从打开仓库到推进一组实验

Agent Skills 把目标、入口、参数和输出文件串起来，让你少花时间翻目录，多花时间做实验。

涉及协议细节时，建议先准备规范文档

如果你的任务涉及协议语义解释、实现补齐或和规范逐条对照，建议先把相关 PDF 放进仓库。这样 Agent 的解释更有依据；如果你只是先跑现成场景，这一步不是前提。

  1. 前往 灵衢官网 (unifiedbus.com) 下载《灵衢基础规范》PDF
  2. 将 PDF 放入项目的 dev-knowledge/specification/ 目录
  3. 如果研究会涉及 CHI，也可以把对应规范放到同一目录

准备好之后，Agent 在解释协议问题或辅助实现时，就能把代码、文档和实验现象放在一起交叉核对。

1

先说明你的研究目标

先把目标说清楚，例如你想比较哪种路由、观察哪类结果、或者理解哪段实现。Agent Skills 会帮助你把问题映射到仓库里的现有入口。

你说："我想看看 Clos 网络里 Pod 间通信的吞吐量表现"
Agent：先定位 `clos_32hosts_pod2pod`，再说明应该如何运行以及输出里该看什么
2

让 Agent 帮你对齐入口和参数

Agent 会帮助你识别现成场景、相关脚本、关键参数和结果目录，避免你一上来就在配置文件和源码之间来回跳。

你说："把带宽改成 200Gbps，跑 10 个流"
Agent：指出这些参数分别落在哪些配置项里，并整理新的运行命令与检查点
3

围绕结果继续追问

仿真跑完后，Agent 可以把 `task_statistics.csv`、`throughput.csv` 和源码中的相关逻辑一起串起来，帮助你判断哪些现象有证据支持，哪些还需要继续验证。

你说："第 3 个流为什么慢了 2 倍？"
Agent：先对比端口吞吐和任务完成时间，再提示可能的拥塞热点与还需补查的证据
4

在人工确认下继续迭代

后续可以让 Agent 帮你整理下一轮实验、定位实现入口、补充验证命令，但策略是否合理、结论是否成立，仍然由研究者自己把关。

你说："换成自适应路由再跑一遍，和刚才的结果对比"
Agent：切到 `routing-adaptive` 相关入口，并按相同指标组织两组结果对比
为什么更准

先查项目事实，再给建议

普通对话常按通用经验回答。Agent Skills 先读仓库规则、定位实验入口、调用项目内 tools、核对输出文件，再给建议；同时复用知识卡里的项目经验与判断规则。这样更容易减少大模型幻觉，也更方便追溯结论依据。

读取项目约束

进入仓库后，先读取规则和说明文件，对齐目录职责、命令口径和修改边界，再继续往下做。

识别现成入口

先把你的问题映射到现成 case、脚本和结果文件，不用每次都从文件树里重新找入口。

优先使用 tools

在仿真配置生成、真实参数查询和输出整理上，优先调用项目内 tools，少靠大模型凭空补全，做事更稳也更快。

解释实验输出

仿真完成后，围绕任务完成时间、链路吞吐等结果文件组织解释，并把判断尽量落到具体输出和代码路径上。

复用知识卡经验

知识卡沉淀项目里的专家经验、稳定结论和排查规则。问到相关问题时，可直接复用这些经验，不必每次从零梳理。

提示风险边界

已有证据和暂时猜测会分开说明，避免把一次结果、示例配置或局部现象直接当成最终结论。

研究记忆

让每次实验的发现都留下来

仿真研究的价值在于不断积累。Agent Skills 可以把每次对话里重要的判断和发现写成知识卡，留给下一轮实验继续复用，不必每次都从零交代背景。

重要发现可以留存

每次实验里形成的关键结论、配置边界和现象规律，都可以写进知识卡，以后随时调出来用，不会随着对话窗口关闭而消失。

新实验关联旧经验

下一轮实验开始时，当前问题可以先和已有知识卡对上，之前记下的关键线索能直接继续用，不需要重新交代背景。

知识卡会持续充实

随着知识卡持续充实，后续对话能直接复用更多项目经验；新的稳定结论也可以继续写回知识卡。

常见问题

使用前须知

我完全不懂 ns-3 或 C++，能用吗？

可以。先拿它找场景、看结果、理目录，进入项目会快很多；后面再慢慢补自己的工程判断就行。

Agent Skills 和普通的 AI 对话有什么区别？

区别在于它不只靠大模型临场回答，还会先读规则、调 tools、复用知识卡，再结合实验入口、参数语义和结果文件给建议。

支持哪些 AI 工具？

当前主要面向 Cursor、Claude Code 和 Codex 这类能读取仓库规则的工具。打开仓库后，它们会通过仓库内的技能文件和相关规则直接接入这套能力。

我能添加自己的规则吗？

可以。你可以在仓库里补充或修改相关技能文件和规则，把实验习惯、目录职责和团队约定写进去，让工具在更局部的范围内遵循这些约束。

仿真结果可靠吗？

仿真结果是否可靠，最终还是看平台建模精度、参数设置和验证过程。Agent Skills 主要帮助你少走弯路，把实验组织、参数定位和结果解读做得更清楚。

把 AI 变成项目内助手

先从现成场景开始，再让它帮你对参数、跑实验、读结果，把每轮实验的发现继续接下去。
